## 🛠 حل مشکل کرش و بسته شدن ناگهانی برنامه
اگر اپلیکیشن شما روی شبیه‌ساز یا گوشی واقعی اندروید بلافاصله پس از باز شدن **بسته می‌شود (Crash on Launch)**، دلیل آن دو خطای فوق‌العاده حساس است:

1. **تلاش برای اجرای Pip درون اندروید:**
   فایل‌های فلت پایتون که از کلاس ترمینالی و کدهای خودکار نصب پکیج استفاده می‌کنند، با اجرای دستور `subprocess` در سیستم‌عامل اندروید کرش می‌کنند. دسترسی ترمینال لایه پایتون اندروید محدود بوده و با اجرای فرآیند فرزند پایتون فوراً بلاک گشته و کل رشته (Thread) اصلی از کار می‌افتد.
   * **راه‌حل اعمال‌شده:** ما ساختار پردازشی پایتون را شناسایی کرده‌ایم تا این سیستم نصب اتوماتیک را دقیقاً فقط در محیط تست/سرور اجرا کند و روی کلاینتِ موبایلی آن را کاملاً حذف کند.

2. **خطای ModuleNotFoundError برای httpx یا پکیج‌های مشابه:**
   فلت از موتور `serious_python` برای کاندنس کردن یادداشت‌ها و ساختار کدهای اجرایی استفاده می‌کند. اگر پکیجی از طریق کد بالا استفاده شده ولی به درستی درون `requirements.txt` در زمان کامپایل قرار نگرفته باشد، در اولین میلی‌ثانیه‌ی ایمپورت مستقیم در لایه اندروید با خطا مواجه شده و چون کلاینت هیچ پنجره باز یا فیلتری برای نمایش خطای پایتون ندارد، کلاینت درجا ترمینیت می‌شود.

## 📦 تنظیمات طلایی فایل `requirements.txt`
برای اینکه هم فرآیند بیلد گیت‌هاب موفقیت‌آمیز باشد و هم موتور پایتون داخل APK دقیقاً پکیج‌های مکمل اندروید را شناسایی کند، نباید کلمه‌ی کلیدی `flet` را به تنهایی درون نیازمندی‌ها بنویسید (زیرا پکیج فلت دسکتاپ تلاش می‌کند ابزار سنگین مانیتورینگ سیستم `watchdog` را نصب کند که نسخه‌ی کامپایل‌شده‌ی C آن برای معماری پردازنده گوشی `arm64-v8a` یافت نمی‌شود).

محتوای فایل **`requirements.txt`** روت پروژه را دقیقاً به شکل زیر اصلاح کنید:

```text
# 📌 فقط پکیج‌های مورد نیاز منطق فرانت-اند/بک‌اند پایتون بدون هسته فرعی سیستم دایلر دسکتاپ:
httpx==0.27.0
requests==2.31.0
urllib3==2.1.0
certifi==2024.2.2
```

> **🔑 نکته کلیدی:** به هیچ عنوان نیازی به نوشتن خود پکیج `flet` درون فایل `requirements.txt` وجود ندارد! زیرا انجین بیلدر فلت اندروید (`flet build apk`) در بدنه متادیتای سیستم فلاتر خود فلت را لود و به صورت بومی سینک می‌کند. قرار دادن فلت دسکتاپ در این فایل باعث دریافت ارورهای سنگین عدم تطابق پتلفرم نظیر ارور `watchdog` می‌شود.

## ✨ ویژگی‌ها و امکانات کلیدی اپلیکیشن
این پروژه تلاشی بی‌نظیر برای مدیریت فایل و پلتفرم یادداشت‌نگاری ابسیدین روی موبایل است:

* **⚡️ مفسر فوق‌سریع Markdown:** رندرینگ لحظه‌ای فرمت‌های پیشرفته فایل‌های متنی Markdown بدون کوچک‌ترین افت فریم.
* **📂 همگام‌ساز دو طرفه (Bi-directional Sync):** به کمک وب‌سرویس‌ها مجهز به کانکشن امن کلاینت جهت همگام‌سازی یادداشت‌ها با سرورهای لوکال یا ابری.
* **🎨 تم تاریک کیهانی و رابط بصری مدرن:** طراحی متریال فوق حرفه‌ای مجهز به افکت‌های حرکتی نرم و استایلیش همراه با پشتیبانی از فونت وزیرمتن و شبکه‌های هم‌ترازی نوین.
* **🔍 جستجوی درختی قدرتمند:** جستجو میان متون به صورت آفلاین بدون نیاز به اتصال دائم اینترنت.

## 🚀 راهنمای کامل بیلد با GitHub Actions
برای بیلد بی‌دردسر این فایل پروژه از روی کدهای منبع به فرمت APK رسمی اندروید، مراحل زیر کارساز خواهد بود:

### ۱. ساختار گردش‌کار گیت‌هاب (فایل `.github/workflows/flet-apk.yml`)
مطمئن شوید فایل ساختار بیلد شما مانیفست زیر را کامل داشته باشد:

```yaml
name: Build Flet Android APK
on:
  push:
    branches: [ main ]
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Install Python 3.12
        uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Setup Flutter Toolchain
        uses: subosito/flutter-action@v2
        with:
          channel: 'stable'

      - name: Install Android SDK
        uses: android-actions/setup-android@v3

      - name: Install Build Engine Tools
        run: |
          pip install --upgrade pip
          pip install flet

      - name: Build Android APK
        run: |
          # پذیرش گواهینامه‌های رسمی اندروید پیش از کامپایل
          yes | sdkmanager --licenses || true
          flet build apk --verbose

      - name: Upload Artifact APK
        uses: actions/upload-artifact@v4
        with:
          name: obsidian-flet-app
          path: build/apk/*.apk
```

## 🔒 نکات امنیتی و بهینه‌سازی عملکرد در اندروید

1. **انتقال درخواست‌های شبکه به لایه امن:**
   در کدنویسی‌های HTTP ترجیحاً همواره از کتابخانه‌ی سبک `httpx` و حالت همگام‌ساز پاسخ استفاده کنید که برای کار با اندروید بهینه‌تر از ترد قدیمی `requests` است.

2. **محافظت از فایل‌های ذخیره‌شده (Local Vault):**
   اندروید فایل‌های شخصی کاربر را در دایرکتوری سندباکس اپلیکیشن نگه می‌دارد. اطلاعات دایرکتوری را با کدهای پایتونی زیر به مسیر استاندارد اسناد هدایت کنید:
   ```python
   import os
   # دریافت دایرکتوری امن برنامه به صورت کراس پلتفرم
   data_dir = os.getenv("ANDROID_DATA_DIR") or os.path.expanduser("~")
   ```
